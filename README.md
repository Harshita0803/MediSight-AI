# MediSight-AI — Auditing LLM Explanations of Clinical Readmission Risk

(https://doi.org/10.5281/zenodo.21183162)

A reproducible pipeline for 30-day hospital readmission prediction **and** a validated
protocol for auditing whether large-language-model (LLM) explanations of those predictions
are actually *faithful* to the model — or merely fluent.

Everything runs on **open synthetic EHR data**
([Synthea](https://github.com/synthetichealth/synthea)). No credentialed dataset is required.

---

## Research Contribution

Hospitals increasingly pair a structured risk model with an LLM that narrates *why* a patient
was flagged. However, a fluent explanation is not necessarily faithful: the factors the LLM
names may differ from the factors the underlying model actually used.

This repository accompanies the paper:

> **“Fluent but Unfaithful: Auditing Large Language Model Explanations of Clinical
> Readmission Risk Against Model Attributions.”**
> Sharma, H. (2026). Zenodo. https://doi.org/10.5281/zenodo.21183162

The paper introduces a two-condition protocol to measure this gap:

- **Grounded** — the LLM receives the model’s SHAP attributions and is asked to restate them.
- **Reasoning** — the LLM receives only the patient’s raw data and must infer the drivers itself.

### Key Results (n = 200 patients)

| Metric | Grounded | Reasoning | Interpretation |
|---|---|---|---|
| Coverage ↑ | 0.626 | **0.297** | Reasoning names fewer than half the model’s true drivers |
| Fabrication ↓ | 0.000 | **0.565** | Reasoning introduces factors outside the model’s top-k |
| Direction accuracy ↑ | 0.979 | 0.797 | Reasoning contradicts the SHAP sign ~20% of the time |

All text-based metrics were validated against human annotation (mention detection
κ = 0.835; direction raw agreement = 0.844). The results show that **attribution-grounded
explanations are faithful**, while free-reasoning explanations frequently diverge from the
model.

## Reproduce the Faithfulness Study (Paper Results)

### Installation

Requires Python 3.11. This project uses [`uv`](https://github.com/astral-sh/uv):

```bash
git clone https://github.com/Harshita0803/MediSight-AI.git
cd MediSight-AI
uv sync                     # or: pip install -e .
cp .env.example .env        # add your OPENAI_API_KEY (see note below)
```

### Run the faithfulness evaluation

```bash
# Grounded condition (Condition A)
python -m evaluation.run_evaluation --features data/eval_features.csv --n 200 --condition a --llm

# Reasoning condition (Condition B)
python -m evaluation.run_evaluation --features data/eval_features.csv --n 200 --condition b --llm
```

> **Tip:** omit `--llm` to run with a deterministic template-based explainer (no API key required).

### Validate the parser against human annotation

```bash
# Sample explanations for labeling
python -m evaluation.annotate sample \
    --results reports/faithfulness_results_condB.json \
    --n 50 \
    --out reports/annotation_sample.csv

# ... fill in the human_mentioned / human_direction columns in the CSV ...

# Score parser-vs-human agreement (Cohen's kappa)
python -m evaluation.annotate score --sheet reports/annotation_sample.csv
```

The clinical feature lexicon lives in `evaluation/lexicon.py`.

### Full pipeline (optional)

To retrain the models and regenerate everything from scratch:

```bash
python main.py                  # ETL: Synthea CSVs → PostgreSQL
python -m models.train          # train XGBoost + compute SHAP
python -m rag.ingest            # embed clinical notes → local FAISS index
uvicorn api.main:app --reload   # serve predictions + RAG
streamlit run app/streamlit_app.py
```

## Data

All experiments use synthetic patients generated by
[Synthea](https://github.com/synthetichealth/synthea). This enables full reproducibility
without data-access restrictions and clearly bounds the scope of the study: results
characterise a **method** and a divergence **phenomenon**, not the true clinical causes of
readmission.

## Limitations

- **Synthetic data only** — demonstrates a method, not a real clinical finding.
- **Single LLM (`gpt-4o-mini`) and single annotator** — stronger models and a second
  annotator are natural extensions; the harness supports swapping the model.
- **Deterministic lexicon parser** — mention recall ≈ 0.80 caps absolute coverage; the
  contrast between Grounded and Reasoning remains robust because the same parser scores both.
